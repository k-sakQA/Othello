# ホテル予約システム 仕様書

## 1. 概要

本システムは、ユーザーがオンラインでホテルの宿泊プランを予約できるWebアプリケーションです。

## 2. 機能仕様

### 2.1 宿泊予約機能

#### 2.1.1 予約入力画面

**URL**: `https://hotel-example-site.takeyaqa.dev/ja/reserve.html?plan-id=0`

**画面レイアウト**:
- ヘッダー: プランの名称と料金が表示される
- 入力フォーム: 以下の項目を入力
  - 宿泊日（必須）: カレンダーから選択
  - 宿泊数（必須）: 1～9泊から選択
  - 人数（必須）: 1～9名から選択
  - 朝食バイキング: チェックボックス（任意）
  - 昼からチェックインプラン: チェックボックス（任意）
  - お得な観光プラン: チェックボックス（任意）
  - 氏名（必須）: テキスト入力
  - 確認のご連絡（必須）: ドロップダウン（メール/電話/希望しない）
  - ご意見・ご要望: テキストエリア（任意）
- 予約内容を確認するボタン

#### 2.1.2 入力仕様

**宿泊日**:
- 形式: YYYY年M月D日（カレンダーから選択）
- 制約: 当日以降の日付のみ選択可能

**宿泊数**:
- 範囲: 1～9泊
- デフォルト: 1泊

**人数**:
- 範囲: 1～9名
- デフォルト: 1名

**朝食バイキング**:
- オプション料金: 1人1泊あたり1,000円追加

**昼からチェックインプラン**:
- オプション料金: 1,000円追加
- 通常チェックイン時刻: 15:00、このプランでは12:00からチェックイン可能

**お得な観光プラン**:
- オプション料金: 1,500円追加
- 近隣観光施設の割引チケット付き

**氏名**:
- 形式: 全角文字（ひらがな・カタカナ・漢字）
- 文字数: 1～50文字
- 制約: 必須入力

**確認のご連絡**:
- 選択肢: 
  - メール: 予約確認メールを送信
  - 電話: ホテルから電話連絡
  - 希望しない: 連絡なし
- 制約: 必須選択

**ご意見・ご要望**:
- 形式: 任意のテキスト
- 文字数: 0～500文字
- 制約: 任意入力

#### 2.1.3 バリデーション

**必須入力チェック**:
- 宿泊日、宿泊数、人数、氏名、確認のご連絡が未入力の場合、エラーメッセージを表示

**エラーメッセージ**:
- 「このフィールドを入力してください。」（必須項目未入力）
- 「適切な形式で入力してください。」（形式不正）

#### 2.1.4 予約確認画面

**遷移条件**:
- 予約入力画面で「予約内容を確認する」ボタンをクリック
- すべての必須項目が正しく入力されている

**「予約内容を確認する」ボタンをクリックした後の確認画面での表示内容**:
- 入力画面で入力した以下の内容が表示される
  - 入力された予約内容の確認
  - プラン名、料金
  - 宿泊日、宿泊数、人数
  - 選択されたオプション
  - お客様情報（氏名、連絡方法）
  - 合計金額
  - 「この内容で予約する」ボタン（今回のスコープ外）

**合計金額の計算**:
```
基本料金 × 宿泊数 × 人数
+ 朝食バイキング料金（1,000円 × 宿泊数 × 人数）
+ 昼からチェックインプラン料金（1,000円）
+ お得な観光プラン料金（1,500円）
```

## 3. 非機能要件

### 3.1 レスポンシブデザイン

- PC、タブレット、スマートフォンで適切に表示
- 画面幅に応じてレイアウトを調整

### 3.2 アクセシビリティ

- キーボード操作で全機能を利用可能
- スクリーンリーダー対応（適切なaria属性）

## 4. 既知の制約事項

- 予約は当日以降の日付のみ可能
- 同時に複数の部屋を予約することはできない
- キャンセルはシステム外（電話）で対応する（システムのスコープ外）
